# Decisiones — Bahía

Decisiones importantes + el porqué. La más reciente arriba.

- **2026-06-15 · Objetivo #1 = Motor Day Pass → Inscripción.** Es el único flujo con ingreso nuevo
  desde el día 1, 100% atribuible, y el caso de uso perfecto para probar el BI. *Ajuste clave de la
  revisión: medir por incrementalidad vs un grupo de control, metas realistas (5-15%, no 30-45%), y
  el 8% sobre ingreso NETO (− day pass acreditado, − churn <90d).*

- **2026-06-15 · Pivote a capa "companion", NO reemplazar Trainingym.** Reemplazar un SaaS maduro es
  una trampa de capital ($150-400k/año) y un commodity; nuestro foso es la relación + los datos + ser
  el sistema de acción/inteligencia. Entramos por marketing/ingreso (0% solapado con Trainingym).

- **2026-06-15 · NO clonar Trainingym a futuro como objetivo.** El activo replicable es NUESTRA capa
  (copiloto para clubes que ya usan gym-SaaS), no la de ellos. Conviene explorar su partner program.

- **2026-06-15 · Geografía: Bahía de Banderas, Nayarit** (no Puerto Vallarta/Jalisco). Metro
  Vallarta–Riviera Nayarit; el club jala locales de alto poder + expats/snowbirds + turistas.

- **2026-06-13 · Colaboración via repo + handoffs diarios.** El contexto se comparte por git (STATUS +
  handoffs), no por la conversación de Claude. PRs a `dev`, nunca push a `main` directo.
